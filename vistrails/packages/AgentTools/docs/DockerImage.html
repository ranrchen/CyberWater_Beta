DockerImage

 This module simply creates a single Docker object.
All modules that require to execute Docker images will require a DockerImage module to indicate the image needed or needed to pull from Docker hub. To use this module, user needs install the Docker in advance, and start the Docker engine while executing the docker container within the CyberWater.

[INPUTS]
Image:
    Existing Images: Select an existing images from the local machine.
    Pull a new images: Pull a new images from remote docker hub.
GRASS GIS is not installed in the default path on your Operative System C:\Program Files\GRASS GIS 7.2.0\grass72.bat
Volume Mounts:
    Source: Path where the file or directory is stored in the local machine.
    Destination: Path where the file or directory is mounted in the container.

[OUTPUTS]
DockerImage: Provide a connection output port to the Docker engine.
